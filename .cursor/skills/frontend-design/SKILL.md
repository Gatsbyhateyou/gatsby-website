# Frontend Design（前端设计）

引导做出**有辨识度、生产级**的前端界面，避免千篇一律的「AI 味」。用户提供需求（组件、页面、应用或整站），先做设计思考再写可运行代码，注重美学细节与创意选择。

## 何时使用

用户说以下任一内容时使用：

- 做前端、做页面、做落地页、做组件、美化界面、做网站、做仪表盘、做海报
- frontend design、HTML/CSS 布局、React 组件样式、给页面做设计

## Design Thinking（设计思考）

写代码前先明确并**坚定一个美学方向**：

- **Purpose**：界面解决什么问题？谁在用？
- **Tone**：选一种并贯彻：极简、极繁、复古未来、有机/自然、奢华/精致、玩具感、杂志/编辑风、粗野/原始、Art Deco/几何、柔和/ Pastel、工业/实用等。
- **Constraints**：技术栈、性能、无障碍等限制。
- **Differentiation**：让人记住的那一点是什么？

**关键**：方向清晰、执行到位。极繁和极简都可以，重在**有意图**。

然后实现**可运行代码**（HTML/CSS/JS、React、Vue 等），要求：

- 生产可用、功能完整
- 视觉有冲击力、易记住
- 风格统一、有明确美学立场
- 细节经得起推敲

## Frontend Aesthetics Guidelines（前端美学要点）

- **Typography**：用有特色、好看的字体；避免 Arial、Inter 等通用字体；可用一个突出的标题字 + 一个清晰的正文字。
- **Color & Theme**：整体风格统一，用 CSS 变量；主色 + 少量强调色优于平均配色。
- **Motion**：用动画做氛围和微交互；HTML 优先 CSS，React 可用 Motion；重点放在高光时刻（如整页入场、错落出现），再配合滚动、hover。
- **Spatial Composition**：非常规布局、不对称、重叠、斜线、打破网格、大量留白或紧凑密度。
- **Backgrounds & Visual Details**：用渐变、噪点、几何、层次、阴影、边框、颗粒等营造氛围，而非纯色。

**禁止**：Inter/Roboto/Arial、紫白渐变、套路布局和组件、无上下文特色的模板感。

**要求**：每个设计都应有不同气质（明/暗、不同字体、不同风格），不重复同一套「安全选项」。实现复杂度要和美学一致：极繁需要丰富动效与细节，极简需要克制与精准的排版与间距。
