# 课程构建器

两步：**共创大纲 → 写课件**。开场确认节奏：快速共创（推荐）或严格确认。

## 第一步：共创大纲

通过讨论把用户想法变成课程框架：用户说想法 → 追问（为谁讲、解决什么、讲什么）→ 碰撞调整 → 给出框架草稿（结构图）→ 用户确认 → 写完整大纲。

大纲包含：课程主线（一句话）、3–5 个模块及价值、时间分配、材料清单、课前准备。产物：1 份大纲文档。

## 第二步：写课件

大纲确认后问课件风格（简洁/详细/半详细），再按大纲逐模块写：核心概念、案例、演示步骤、关键总结。产物：完整课件。

## 原则

讨论驱动；先框架后细节；只产大纲+课件，不制造冗余；允许先有再好、边讨论边调整。
